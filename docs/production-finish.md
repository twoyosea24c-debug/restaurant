# 公開前仕上げチェック

## 1. 実店舗情報

管理画面の `設定 -> 店舗情報` で次を確認します。

- 店舗名
- 紹介文
- 電話番号
- メールアドレス
- 営業時間
- 定休日

電話番号が `03-0000-0000` のままなら、本番公開前に実番号へ変更します。

## 2. テストデータ整理

管理画面の `公開前チェックリスト` でテストデータ候補を確認します。

削除対象の例:

- `@example.com` の顧客
- `BKG-1001` の予約
- `ORD-1001` の注文
- メモに `サンプル` を含む予約

削除は本番データに影響するため、実行前に必ずバックアップを取得します。

## 3. スマホ実運用テスト

スマホで次の順に確認します。

- 予約を送信し、予約番号が表示される
- 注文を送信し、注文番号が表示される
- 問い合わせを送信し、完了文言が表示される
- 管理画面で予約、注文、問い合わせが見える
- メール通知が届く

## 4. 独自ドメイン

独自ドメインは `docs/domain-setup.md` に沿って設定します。

設定後、Vercelの `NEXT_PUBLIC_APP_URL` も独自ドメインへ変更します。

## 5. 公開文言

公開ページの紹介文、商品説明、注文時の注意文が実店舗の運用に合っているか確認します。

## 6. バックアップ運用

管理画面の `バックアップ/復元` からバックアップJSONを取得できます。

推奨運用:

- 公開直前に1回取得
- メニュー、商品、在庫を大きく変更する前に取得
- 月1回、定期的に保管
